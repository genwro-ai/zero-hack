The problem · why it matters

Does the model learn process logic, or just memorize?

Semiconductor flows follow a hidden grammar: ~198 process steps, 115–150 steps per sequence, a shared backbone plus family-specific steps. A model that truly learns the rules can catch process violations on product families it has never seen — exactly the case that matters on a real fab line. We score four tasks:

  • Task 1 · Next stepRank the operation that should come next.
  • Task 2 · CompletionFinish a valid flow from a partial prefix.
  • Task 3 · AnomalyFlag sequences that break process rules.
  • Task 4 · Out-of-distributionTrain on two families, test on an unseen third.

Approach · 1 / 3 — data & honest evaluation

Generate valid data at scale, evaluate leak-free

01

Grammar generator

A synthetic generator builds rule-valid flows at scale — 100k per family — with the organizer's rule checker as a backstop on every sequence.

02

Leave-one-family-out

Train on two families, hold out the third entirely. ID test draws from the trained families; OOD test is the unseen family — the Task-4 differentiator.

03

Leak-free OOD set

A novel-family generator builds flows from atomic role-typed operations: 0 OOV steps, 0 collisions with real families. Honest OOD, no shortcuts.

Approach · 2 / 3 — key decision: positional encoding

ID can't separate them — OOD picks the winner

Five variants, trained on two families, scored on the unseen third. Blue = in-distribution · orange = out-of-distribution.
GPT-absolute collapses to 0.43 OOD anomaly ROC-AUC — below chance. Learned absolute positions memorize where a step sat in the training families; that signal doesn't transfer. We chose GPT-ALiBi: its distance bias gives the best OOD completion (0.806) and stays stable across unfamiliar prefix lengths — the strongest all-task choice.

What breaks on an
unseen family

Local step transitions look alike across families, so next-step prediction barely moves (top-1 stays ~0.66).

Anomaly detection is what breaks: the ID-tuned threshold flags almost everything, F1 collapses to the all-positive baseline (~0.67), and threshold-free ROC-AUC drops to ~0.77.

Results · 3 / 3 — what worked, what didn't

The wins — and the dead ends, reported straight

✓ Worked

  • Rule-valid generation at scale (100k/family) with the validator as a backstop
  • Plain likelihood is a near-perfect ID anomaly detector (AUC ~1.0)
  • Leak-free novel-family generator → an honest OOD set
  • DPO sharpened ID anomaly: AUC 0.981 → 0.999, F1 0.936 → 0.996

✗ Didn't

  • OOD threshold transfer — the core open problem; F1 → baseline
  • Decode-time rules changed 0 / 2000 top-1 tokens — model already compliant
  • Scheduled sampling didn't beat teacher forcing at this scale
  • Exact-match completion ≈ 0 — long flows, many equally-valid endings
